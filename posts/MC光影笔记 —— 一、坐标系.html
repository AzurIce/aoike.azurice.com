MC光影笔记 —— 一、坐标系

gbuffer阶段的输入顶点gl_Vertex所处的模型坐标系与optifine中的模型坐标系并不相同。

MC模型坐标系

gl_Vertex\LARGE gl\_Vertex gl_Vertex

对于方块而言，是一个像素位于其所在子区块（16x16x16）中的坐标，坐标值取值为[0, 16]且连续。

color = vec4(gl_Vertex.xyz/16, 1);

（对于实体而言似乎与眼坐标有关）

眼坐标系

gl_ModelViewMatrix×gl_Vertex\LARGE gl\_ModelViewMatrix \times gl\_Vertex gl_ModelViewMatrix×gl_Vertex

color = vec4((gl_ModelViewMatrix * gl_Vertex).xyz, 1);

Optifine/shadersmod模型坐标系

gbufferModelViewInverse×gl_ModelView×gl_Vertex\LARGE {\color{blue}gbufferModelViewInverse} \times gl\_ModelView \times gl\_Vertex gbufferModelViewInverse×gl_ModelView×gl_Vertex

color = vec4((gbufferModelViewInverse * gl_ModelViewMatrix * gl_Vertex).xyz, 1);

color = vec4((gbufferModelViewInverse * gl_ModelViewMatrix * gl_Vertex).xyz + cameraPosition, 1);

可以加上cameraPosition得到在MC世界中的坐标系（简称世界坐标系），下图为MC坐标(0, 0, 0)附近。

裁剪坐标系

gbuffer阶段的输出gl_Position和ftransform()便是裁剪坐标系下的坐标。

composite阶段的输出gl_Position和ftransform()

gl_ProjectionMatrix×gl_ModelViewMatrix×gl_Vertex\LARGE gl\_ProjectionMatrix \times gl\_ModelViewMatrix \times gl\_Vertex gl_ProjectionMatrix×gl_ModelViewMatrix×gl_Vertex

gbufferProjection⇔gl_ProjectionMatrix\LARGE {\color{blue}gbufferProjection} \Leftrightarrow gl\_ProjectionMatrix gbufferProjection⇔gl_ProjectionMatrix

// 下面二者相等
color = vec4((gl_ProjectionMatrix * gl_ModelViewMatrix * gl_Vertex).xyz, 1);
color = vec4(ftransform(), 1);

NDC坐标系

color = vec4((ftransform().xyz + 1) / 2, 1);

纹理坐标

color = texcoord;

一、模型坐标系（ModelCoord）—— gl_Vertex

推测：

// gbuffers_textured.vsh
#version 460 compatibility

out vec4 color;

void main() {
    vec4 vertex = gl_Vertex;
    gl_Position = ftransform();
    color = vertex;
}

// gbuffers_textured.fsh
#version 460 compatibility

in vec4 color;

/* DRAWBUFFERS:0 */

void main() {
    gl_FragData[0] = color;
}

将 gbuffers_textured.vsh 中的

color = vertex;

改为

color = vertex / 16;

可以推测，gl_Vertex （模型坐标系）坐标值为该像素位于其所在子区块的坐标，xyz值大小介于[0，16]之间.

验证：

将 gbuffers_textured.vsh 中的

color = vertex / 16;

改为

if(vertex.x == 0)
    color = vec4(1, 1, 1, 1);
else
    color = vec4(0, 0, 0, 1);

vertex.x == 8 || vertex.z == 2 || vertex.y == 4

可以得到：

可以证明推测正确。

结论：

gl_Vertex 存储的是某一像素位于其所在子区块中的坐标，坐标值取值为[0, 16]且连续。

二、归一化的裁剪空间坐标 —— gl_Position

gl_Position 是归一化的裁剪空间坐标，后续用于图元装配、片段着色、光栅化后除以w才转化为NDC坐标，之后进行视口变换变为屏幕坐标。

三、眼坐标系（ViewCoord）

gl_ModelViewMatrix * gl_Vertex

四、“我的世界坐标系” —— optifine/shadermod 的模型坐标系

gbufferModelViewReverse * gl_ModdelViewMatrix * gl_Vertex

以相机位置为原点的一种世界坐标系。

可将其加上cameraPosition 得到世界坐标系。
