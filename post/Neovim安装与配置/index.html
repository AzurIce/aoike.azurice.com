插件

一、插件管理器 dein.vim

dein 的 github 页面： Shougo/dein.vim: Dark powered Vim/Neovim plugin manager (github.com)

在使用 dein 之前，必须先定义安装目录，安装目录取决于个人，但不能是vim的插件目录。

1. 安装 dein.vim

dein.vim 的安装方式是运行一个官方的脚本，并传入安装目录。

  • [Windows]

    # 下载脚本 并存为 installer.sh
curl https://raw.githubusercontent.com/Shougo/dein.vim/master/bin/installer.sh > installer.sh

# 运行脚本 传入安装目录 ~/.config/nvim/dein
sh ./installer.sh ~/.config/nvim/dein

  • [Linux]

    # 下载脚本 并存为 installer.ps1
iwr https://raw.githubusercontent.com/Shougo/dein.vim/master/bin/installer.ps1 -OutFile installer.ps1

# 运行脚本 传入安装目录 ~/.config/nvim/dein
./installer.ps1 $env:LOCALAPPDATA/nvim/dein

而更本质上，官方的脚本其实做的也就是将目录从 github 上 clone 下来。

三、markdown-preview

官方 github 仓库：iamcco/markdown-preview.nvim: markdown preview plugin for (neo)vim (github.com)

安装部分：

要注意，一定要先安装 Yarn。

call dein#add('iamcco/markdown-preview.nvim', {'on_ft': ['markdown', 'pandoc.markdown', 'rmd'],
					\ 'build': 'sh -c "cd app && yarn install"' })

配置部分：

" 是否在打开 Markdown 文件时自动打开预览
" default: 0
let g:mkdp_auto_start = 0

" 是否在切换到非 Markdown 文件或退出编辑 Markdown 文件时自动退出预览
" default: 1
let g:mkdp_auto_close = 1

" 是否使用慢刷新
" 慢刷新：只有在保存或离开插入模式时才刷新预览
" 非慢刷新：在编辑或移动指针时刷新预览
" default: 0
let g:mkdp_refresh_slow = 0

" 是否令 MarkdownPreview 对所有文件可用（默认只能在 Markdown 文件中使用）
" default: 0
let g:mkdp_command_for_global = 0

" 是否使预览网页的服务器对网络开放（默认监听 localhost）
" default: 0
let g:mkdp_open_to_the_world = 0

" 打开预览页面的自定义 IP
" useful when you work in remote vim and preview on local browser
" more detail see: https://github.com/iamcco/markdown-preview.nvim/pull/9
" default: ''
let g:mkdp_open_ip = ''

" 打开预览页面的指定浏览器
" default: ''
" =======> 这里我选择 chromium（需要先安装），体积小速度快
let g:mkdp_browser = 'chromium'

" 是否在命令行显示预览页面的 url
" default: 0
let g:mkdp_echo_preview_url = 0

" 用来打开预览页面的一个自定义的 vim 函数名
" 这个函数接收 url 作为参数
" default: ''
let g:mkdp_browserfunc = ''

" Markdown 渲染器的选项
" mkit: markdown-it options for render
" katex: katex options for math
" uml: markdown-it-plantuml options
" maid: mermaid options
" disable_sync_scroll: if disable sync scroll, default 0
" sync_scroll_type: 'middle', 'top' or 'relative', default value is 'middle'
"   middle: mean the cursor position alway show at the middle of the preview page
"   top: mean the vim top viewport alway show at the top of the preview page
"   relative: mean the cursor position alway show at the relative positon of the preview page
" hide_yaml_meta: if hide yaml metadata, default is 1
" sequence_diagrams: js-sequence-diagrams options
" content_editable: if enable content editable for preview page, default: v:false
" disable_filename: if disable filename header for preview page, default: 0
let g:mkdp_preview_options = {
    \ 'mkit': {},
    \ 'katex': {},
    \ 'uml': {},
    \ 'maid': {},
    \ 'disable_sync_scroll': 0,
    \ 'sync_scroll_type': 'middle',
    \ 'hide_yaml_meta': 1,
    \ 'sequence_diagrams': {},
    \ 'flowchart_diagrams': {},
    \ 'content_editable': v:false,
    \ 'disable_filename': 0
    \ }

" 使用自定义的 Markdown 风格（绝对路径）
" like '/Users/username/markdown.css' or expand('~/markdown.css')
let g:mkdp_markdown_css = ''

" 使用自定义的 高亮 风格（绝对路径）
" like '/Users/username/highlight.css' or expand('~/highlight.css')
let g:mkdp_highlight_css = ''

" 自定义预览服务器运行的端口（留空则为随机）
let g:mkdp_port = ''

" 预览页面标题
" ${name} 会被文件名替换
let g:mkdp_page_title = '「${name}」'

" 识别的文件类型（在这些文件类型中 MarkdownPreview 等指令为可用）
let g:mkdp_filetypes = ['markdown']
